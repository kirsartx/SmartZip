# SmartZip 3.6 Kirs.2 解压可靠性设计

**日期：** 2026-07-20
**状态：** 已获用户口头批准，等待书面规格复核
**目标版本：** SmartZip 3.6 Kirs.2 (22)
**目标标签：** `v3.6-kirs.2`

## 1. 背景与已确认问题

用户曾在旧版 SmartZip 解压部分 7z 文件时看到 `Headers Error`。受控测试使用
`C:\Tool\7-Zip-Zstandard\7z.exe`（7-Zip 26.02 ZS v1.5.7 R1）确认：

- 正常 7z：退出码 0；
- 加密头未提供密码：请求密码；
- 加密头密码错误：退出码 2，同时出现 `Wrong password?` 与 `Headers Error`；
- 末端文件头损坏：退出码 2，出现 `Headers Error`；
- 文件截断：退出码 2，出现 `Unexpected end of archive`；
- 非首个 `.7z.002` 分卷：退出码 2，无法作为压缩包打开；
- 首个 `.7z.001` 分卷且其他卷齐全：退出码 0；
- 归档尾部带额外数据：可能退出码 0，但输出警告。

因此，`Headers Error` 不能单独解释为文件损坏，必须结合密码、分卷和其他诊断行分类。

当前实现还有一个已复现的安全缺陷：

- `IsSuccess()` 会在 7-Zip 返回非零退出码时，根据临时目录大小与源压缩包大小之比推断成功；
- 受控 CRC 损坏样本返回退出码 2，但输出比例为 99.87%，现有逻辑仍判定成功；
- 用户当前配置启用 `delSource=1`，错误结果可能触发源包回收；
- 嵌套路径中还存在永久删除临时嵌套源包的路径。

本设计首先消除错误成功和误删风险，然后改善错误提示与恢复操作。

## 3. 架构

### 3.1 模块边界

新增纯诊断模块 `lib/ArchiveDiagnostics.ahk`，由 Ahk2Exe 在编译时打包：

- `ArchiveStatus`：状态常量；
- `ArchiveResult`：统一结果对象；
- `Classify7zResult(stage, exitCode, output)`：无副作用分类；
- `DetectVolumeGroup(path, siblingNames)`：分卷组识别；
- `RedactDiagnostic(text)`：密码和敏感参数脱敏。

`SmartZip.ahk` 继续负责：

- 设置读取和迁移；
- 解压状态机编排；
- 7-Zip 进程调用；
- 临时目录、目标目录和源包生命周期；
- 密码及诊断窗口；
- 批量结果汇总。

最终仍只发布单个 `SmartZip.exe`，不向部署目录增加运行时依赖。

### 3.2 统一结果对象

每个阶段返回 `ArchiveResult`，至少包含：

```text
status
stage
exitCode
archivePath
archiveType
passwordUsed
volumeFirst
missingVolumes
warningLines
errorLines
tempOutputDir
partialOutputDir
isCleanSuccess
mayDeleteSource
```

`passwordUsed` 仅用于内存中的后续解压和密码保存判断，不得进入日志、复制诊断文本或异常消息。

## 4. 状态集合与分类优先级

### 4.1 状态集合

- `OK`
- `OK_WITH_WARNING`
- `NEED_PASSWORD`
- `WRONG_PASSWORD`
- `MISSING_VOLUME`
- `NOT_ARCHIVE`
- `UNSUPPORTED_METHOD`
- `HEADER_CORRUPT`
- `TRUNCATED`
- `DATA_CORRUPT`
- `CANCELLED`
- `IO_ERROR`
- `UNKNOWN_ERROR`

### 4.2 分类优先级

分类读取完整 stdout、stderr 和最终退出码，不再在发现第一条匹配行时杀死检查进程。优先级固定为：

1. 用户取消或退出码 255 → `CANCELLED`
2. 缺卷或无法访问卷 → `MISSING_VOLUME`
3. 需要密码 → `NEED_PASSWORD`
4. `Wrong password` / `Cannot open encrypted archive` → `WRONG_PASSWORD`
5. 不支持的方法或格式特性 → `UNSUPPORTED_METHOD`
6. `Unexpected end of archive` → `TRUNCATED`
7. `Headers Error` 且没有密码或缺卷证据 → `HEADER_CORRUPT`
8. `CRC Failed` / `Data Error` → `DATA_CORRUPT`
9. 无法识别为压缩包 → `NOT_ARCHIVE`
10. 退出码 0 且存在警告 → `OK_WITH_WARNING`
11. 退出码 0 且无警告 → `OK`
12. 其他非零退出码：出现访问拒绝、磁盘空间不足、路径不可用或文件读写失败标记时 → `IO_ERROR`；否则 → `UNKNOWN_ERROR`

同一次输出出现多个标记时使用最高优先级状态，同时保留其他诊断行用于详细信息。

## 5. 解压状态机

每个顶层输入按下列顺序处理：

1. **规范化路径**
   - 确认文件存在；
   - 解析工作目录和目标目录；
   - 不根据扩展名直接断定真实格式。

2. **分卷解析**
   - 识别分卷组；
   - 若选择非首卷且首卷存在，自动重定向到首卷；
   - 批量选择同组多个卷时只生成一个任务；
   - 首卷或中间卷缺失时停止并返回 `MISSING_VOLUME`。

3. **轻量探测**
   - 使用 `7z.exe l -slt -bso1 -bse1 -bsp0 -sccUTF-8`；
   - 完整捕获 stdout、stderr 和退出码；
   - 获取真实格式、加密状态、分卷状态和基础错误。

4. **密码处理**
   - 对需要密码的归档依次尝试去重后的候选密码；
   - 全部失败后显示 SmartZip 自己的密码窗口；
   - 用户可选择“本次使用”或“使用并保存”；
   - 用户取消返回 `CANCELLED`。

5. **完整测试**
   - `test=1` 时，正式解压前执行 `7z.exe t -bso1 -bse1 -bsp0 -sccUTF-8`；
   - 即使 `test=0`，只要本次可能依据 `delSource` 或 `delWhenHasPass` 处理源包，也必须执行完整测试；
   - 测试为 `OK_WITH_WARNING` 时允许继续解压，但本次源包处理被永久关闭；
   - 测试为其他非 `OK` 状态时停止正式解压。

6. **解压到隔离临时目录**
   - 保留现有 7zG 大文件进度和暂停体验；
   - 解压前的检查统一使用 `7z.exe`；
   - 7zG 返回非零时再次使用控制台测试获取可分类诊断；
   - 所有覆盖冲突继续采用现有自动改名语义。

7. **验证**
   - 只有 7-Zip 退出码 0、必要测试为 `OK` 且没有错误标记时，结果为 `OK`；
   - 不再读取或使用 `successPercent` 推断成功；
   - 有警告但没有错误时为 `OK_WITH_WARNING`。

8. **提交输出**
   - `OK`：按现有单文件/单目录/多文件整理规则移动到正式目标；
   - `OK_WITH_WARNING`：移动正常输出，但保留源包并显示警告；
   - 失败且临时目录含文件：整体移动到“不完整”目录；
   - 失败且临时目录为空：删除空临时目录。

9. **处理源包**
   - 仅探测、必要测试和正式解压三个相关阶段均为 `OK`，且 `mayDeleteSource=true` 时处理；
   - 顶层源包只进入回收站，不永久删除；
   - 分卷组不自动删除任何卷；
   - 失败、警告、取消和未知状态一律保留源包；
   - 嵌套归档只有自身状态为 `OK` 时才可从隔离临时目录移除。

## 6. 部分输出目录

失败但存在输出时，目录命名为：

```text
<压缩包名>_解压不完整_<yyyyMMdd-HHmmss>
```

规则：

- 位于用户本次预期的目标目录；
- 使用现有 `PathDupl` 规则避免覆盖；
- 不与成功输出混合；
- 包含一个 UTF-8 文本 `SmartZip-诊断.txt`；
- 诊断文本包含状态、阶段、7-Zip 版本、退出码、缺卷列表和脱敏错误摘要；
- 不包含密码、完整命令行密码参数或剪贴板内容。

## 7. 分卷支持

至少识别：

- `name.7z.001`
- `name.zip.001`
- `name.001`
- `name.part01.rar`
- `name.part10.rar`
- `name.rar` + `name.r00` / `name.r01`

行为：

- 非首卷选择自动重定向到首卷；
- 缺少首卷时提供“定位首卷”按钮；
- 可推导出连续卷号时列出缺失卷；
- 无法可靠推导时只说明“分卷集合不完整”，不伪造卷号；
- 一组分卷在批量任务中只执行一次；
- 所有卷始终保留。

## 8. 嵌套格式识别

- 顶层用户明确选择的文件始终允许 7-Zip 轻量探测，不受扩展名限制；
- 嵌套文件先通过精确扩展名或合法分卷模式成为候选；
- 候选在实际解压前仍必须通过 7-Zip 轻量探测；
- 空扩展名不再自动视为压缩包；
- 迁移时仅删除 `extExp` 中完全等于 `zi`、`7`、`z` 的旧宽泛规则；
- `^\d+$` 和其他用户自定义规则保留；
- 用户自定义规则只能生成“探测候选”，不能绕过 7-Zip 真实性检查。

## 9. 密码体验

候选顺序：

1. 最后一次成功密码；
2. 合法长度的剪贴板文本；
3. 已保存密码（按动态使用次数排序）；
4. 启用时的父目录名称。

要求：

- 去重后再尝试；
- 空密码只测试一次；
- 自动尝试不弹出连续 7-Zip 窗口；
- 全部失败后显示 SmartZip 密码窗口；
- 输入框默认遮罩；
- “使用并保存”沿用现有密码排序和自动清理策略；
- `HEADER_CORRUPT`、`MISSING_VOLUME` 等非密码状态不得继续遍历密码列表。

## 10. 诊断窗口

### 10.1 单任务

标题为 `SmartZip 未完成解压` 或 `SmartZip 解压警告`，显示：

- 文件名；
- 中文原因；
- 推荐动作；
- 是否保留源包；
- 是否存在部分输出。

按状态提供适用按钮：

- 打开部分文件目录；
- 重新输入密码；
- 定位首卷；
- 使用 7-Zip 打开；
- 复制脱敏诊断信息；
- 关闭。

### 10.2 批量任务

- 不中断后续可继续任务；
- 最后显示一次汇总窗口；
- 分为成功、警告、失败、跳过；
- 每项可打开对应输出或复制诊断；
- 不为每个失败项连续弹窗，密码输入和必须选择首卷的交互除外。

## 11. 日志

新增错误诊断日志，默认仅记录警告和失败：

```text
SmartZip-diagnostics.log
SmartZip-diagnostics.log.1
SmartZip-diagnostics.log.2
```

规则：

- 单文件达到 1 MiB 时轮转；
- 最多保留 3 个文件；
- 记录时间、SmartZip 版本、7-Zip 版本、阶段、状态、退出码和错误摘要；
- 本地日志可记录完整文件路径；
- 所有 `-p...` 参数替换为 `-p***`；
- 不记录密码候选、密码输入框内容或剪贴板原文；
- “复制诊断信息”默认只含文件名，不含完整路径。

现有 `cmdLog` 继续作为高级调试选项，但同样必须执行密码脱敏。

## 12. 设置与迁移

- `buildVersion := 22`
- `edition := "Kirs.2"`
- Ahk2Exe FileVersion 保持 `3.6`
- Ahk2Exe ProductVersion 改为 `22`
- `successPercent` 键保留但标记为废弃，运行时不再读取；
- 设置界面移除“判断解压成功百分比”可编辑控件；
- `test` 继续控制普通任务是否预先完整测试，但不能绕过删除前测试；
- 保留用户现有 `zipDir`、密码、目标目录、删除、嵌套和菜单设置；
- 迁移前不重写整个 INI，只更新版本和精确命中的旧宽泛 `extExp` 项。

## 13. 测试

### 13.1 纯分类测试

对 `Classify7zResult` 使用固定输出样本覆盖全部状态，并验证：

- 密码错误优先于同一输出中的 `Headers Error`；
- 缺卷优先于一般文件头错误；
- 退出码 0 加警告得到 `OK_WITH_WARNING`；
- 未识别非零退出码得到 `UNKNOWN_ERROR`；
- 脱敏后不含任何测试密码。

### 13.2 分卷测试

覆盖：

- `.7z.001` 首卷；
- `.7z.002` 自动重定向；
- 中间卷缺失；
- `.part01.rar` 与 `.part10.rar`；
- `.r00` 形式；
- 批量选择同组多卷去重。

### 13.3 真实 7-Zip 集成测试

测试在 TEMP 下创建或变异样本，不接触用户文件：

- 正常 7z；
- 加密头：正确密码、错误密码、取消；
- 损坏末端文件头；
- 截断归档；
- CRC/数据部分损坏；
- 完整和缺失分卷；
- 尾部额外数据警告；
- 扩展名为 `.7z` 的普通文件；
- 无扩展名普通文件。

### 13.4 生命周期安全测试

每个非 `OK` 状态都必须断言：

- 源包仍存在；
- 分卷全部仍存在；
- 不发生永久删除；
- 部分输出仅出现在“不完整”目录；
- 正式成功目录没有混入失败输出；
- 日志和诊断文本不包含密码。

需要保留现有 69 项静态测试全部通过，并新增针对状态机、删除门、宽泛扩展规则移除及版本信息的测试。

## 15. 验收标准

- CRC 部分损坏样本不再被判定成功；
- 任何警告或失败均不会处理源包；
- 加密头错误密码不会被误报为普通文件头损坏；
- 缺卷、截断、文件头损坏和数据损坏显示不同中文原因；
- 非首卷可自动定位首卷；
- 失败的部分输出与正常输出隔离；
- 批量失败只显示一次汇总；
- 密码不出现在任何日志或复制诊断中；
- 现有压缩、上下文菜单、INI 主设置和 7zG 大文件进度体验保持兼容；
- 所有静态、分类、分卷、集成、生命周期和部署测试通过；
- `v3.6-kirs.2` 与 Kirs.1 Release 并存。
