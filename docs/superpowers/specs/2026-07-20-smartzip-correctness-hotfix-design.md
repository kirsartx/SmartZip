# SmartZip 正确性热修设计

日期：2026-07-20
目标分支：`codex/optimize-smartzip`

## 背景

SmartZip 3.4 的核心功能集中在 `SmartZip.ahk`。现有实现中有五项能够从代码与文档直接确认的正确性问题：默认配置阻止单条目嵌套解压、混合文件与目录压缩时路径变量未初始化、压缩格式映射查询使用了错误对象、扩展名子串匹配产生误判，以及智能解压右键菜单状态读取使用了错误的键名大小写。

本轮采用小范围热修，恢复既有功能的预期行为，不改变程序架构或产品交互。

## 范围

本轮只修改 `SmartZip.ahk` 中以下逻辑：

1. `Unzip` 的单条目嵌套解压门控。
2. `CreateZip` 的混合文件与目录参数拼接。
3. `IsArchive` 的普通扩展名识别。
4. 设置界面对 `IsContextMenuVisible` 的智能解压键名调用。

审查中编号 C3 和 C4 都由 `IsArchive` 的一次修正解决，因此范围包含五项缺陷、四处逻辑区域。

## 非目标

- 不拆分 `SmartZip.ahk`，不重构 GUI 或 7-Zip 调用层。
- 不改变 INI 文件结构、默认值、命令行参数或右键菜单名称。
- 不扩充分卷压缩格式，不修改密码存储、排序或日志策略。
- 不改变空扩展名和正则扩展名 `extExp` 的既有处理。
- 不引入新的运行时依赖、安装步骤或自动配置迁移。

## 详细设计

### 1. `Unzip`：按输出类型分别执行嵌套策略

当前单条目分支在移动解压结果后使用：

```ahk
if !this.nesting || !this.nestingMuilt
    continue
```

该条件要求两个开关同时开启，导致默认的 `nesting=1`、`nestingMuilt=0` 无法继续解压单个嵌套压缩文件。

修改后不再使用共享的提前跳过条件，而是保留现有的按类型判断：

- 解压结果是文件时，仅由 `this.nesting` 控制是否调用 `UnZipNesting(outFile, ext)`。
- 解压结果是目录时，仅由 `this.nestingMuilt` 控制是否遍历目录中的文件并调用 `UnZipNesting`。
- 两个开关均关闭时，两条分支自然都不执行。

已确认的产品语义是：`nesting=1` 且 `nestingMuilt=0` 时，单条目嵌套压缩文件仍自动继续解压。目录型多条目嵌套仍需 `nestingMuilt=1`。

### 2. `CreateZip`：初始化混合输入路径

在“文件与文件夹混合”分支进入循环前增加局部变量初始化：

```ahk
path := ""
```

随后沿用现有循环，将每个输入路径按原格式加引号并拼接，再传给 `Run7z`。这只消除 AutoHotkey v2 对未赋值变量执行 `.=` 时的异常，不改变参数顺序、引号格式、输出文件名或 7-Zip 参数。

### 3. `IsArchive`：普通扩展名采用精确映射查询

输入扩展名继续先通过 `StrLower` 规范化。`ini.ReadLoop("ext", this.ext, true)` 已将配置中的普通扩展名以小写键写入 `this.ext`，因此普通格式只需：

```ahk
if this.ext.Has(ext)
    return true
```

删除错误的 `this.ext.Has(zip)` 查询和后续 `InStr(i, ext)` 子串扫描。这样 `.zip` 等完整配置项仍能命中，而 `.z`、`.ip` 等仅为已配置扩展名子串的值不再被误判。

空扩展名仍按当前行为返回 `true`；普通映射未命中后，仍逐项使用 `extExp` 正则规则判断；全部未命中时返回 `false`。

### 4. `IsContextMenuVisible`：传入实际注册表键名

设置界面创建“智能解压”复选框时，将调用参数从 `"unZip"` 改为 `"UnZip"`，与以下既有实现保持一致：

- `IsContextMenuVisible` 用 `"UnZip"` 判断应读取文件类型注册表路径。
- `ContextMenu` 在 `keyPathForFile "\UnZip"` 下创建、读取和删除智能解压菜单。

其他菜单键 `openZip`、`addZip` 和 `unZipCP` 不在本轮改名；其现有路径选择和注册逻辑保持不变。

## 兼容边界

- GUI 布局、文案和事件绑定保持不变。
- `x`、`xc`、`o`、`a` 命令行入口及参数格式保持不变。
- `SmartZip.ini` 的所有键名和默认值保持不变。
- 单文件平铺、多文件进入同名目录、全文件夹分别压缩等现有输出布局保持不变。
- 已配置的精确扩展名和 `extExp` 正则继续生效；此前依赖普通扩展名子串误匹配的未配置后缀将不再被识别。
- 右键菜单注册表位置保持不变，只修复设置页的状态读取。

## 错误处理

本轮不增加新的错误类型或提示框。修复后的逻辑继续使用现有的 `Run7z`、日志、移动与回收错误处理。

- 混合压缩仍由 `Run7z` 报告 7-Zip 执行失败。
- 嵌套解压仍由 `UnZipNesting` 和现有解压流程处理无效或损坏的压缩包。
- 注册表读取继续使用 `RegRead(..., "", "")` 的默认空值；键不存在时复选框显示未注册。
- 扩展名未命中普通映射或正则规则时，`IsArchive` 明确返回 `false`。

1. T1、T2、T5、T8、T9、T11 和 T12 全部通过。
2. T3、T4、T6、T7、T10 和 T13 未出现行为回归。
3. 代码差异只涉及本规格列出的四处逻辑区域及相应测试或验证材料。
4. 不新增配置项、依赖或用户迁移步骤。
5. AutoHotkey v2 语法检查通过；在具备 AutoHotkey v2 与 7-Zip 的 Windows 环境中完成可执行场景验证。
